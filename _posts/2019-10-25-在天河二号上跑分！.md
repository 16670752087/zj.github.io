#### `mp_linpack.slurm`

跑通这一项用到的调度脚本，申请了4个节点，每个节点使用二十核。考虑四个节点共80核，这里节点间消息传递（MPI），节点内共享内存（OpenMP）。

```bash
#!/bin/bash
#SBATCH -J WuK_mp_linpack              # 任务名
#SBATCH -N 4                     # 申请 4 个节点
#SBATCH --ntasks-per-node=1     # 每个节点开 1 个进程
#SBATCH --cpus-per-task=20        # 每个进程占用 20 个 core
module add intelcompiler/18.0.0  # 添加 intelcompiler/18.0.0 模块
export OMP_NUM_THREADS=20         # 设置全局 OpenMP 线程为 20
./build.sh
numactl --interleave=all yhrun ./xhpl
```

#### `hpcg.slurm`

与上一问不同，发现这一问每个节点设置两个进程各十个线程比较合适，否则跑出来的结果会下降到`45GFLOP/s`，猜测HPCG测试中不同CPU之间共享内存的效率很低。

```bash
#!/bin/bash
#SBATCH -J WuK_hpcg              # 任务名
#SBATCH -N 4                     # 申请 4 个节点
#SBATCH --ntasks-per-node=2     # 每个节点开 2 个进程
#SBATCH --cpus-per-task=10        # 每个进程占用 10 个 core
module add intelcompiler/18.0.0  # 添加 intelcompiler/18.0.0 模块
export OMP_NUM_THREADS=10         # 设置全局 OpenMP 线程为 10
./configure WuK
make -j
cd bin
yhrun ./xhpcg_avx2
```
